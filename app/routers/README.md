# Структура роутеров

## Порядок обработки

### admin — роутеры админ-панели (с AdminMiddleware)

1.1. `admin_main.py` — основные админские команды, вход в админ-панель, /promote
1.2. `bookings.py` — управление записями клиентов
1.3. `captains.py` — управление капитанами (назначение, график, зарплата)
1.4. `client_redaction.py` — редактирование данных клиентов
1.5. `clients.py` — управление клиентами (поиск, просмотр, история)
1.6. `create_booking.py` — создание бронирования администратором
1.7. `excursions.py` — управление видами экскурсий
1.8. `finances.py` — управление финансами
1.9. `new_slot.py` — добавление нового слота в расписание
1.10. `notification.py` — отправка уведомлений (массовые и персональные)
1.11. `promocode_edit.py` — редактирование промокодов
1.12. `promocodes.py` — управление промокодами
1.13. `refunds.py` — управление возвратами
1.14. `schedule.py` — управление расписанием экскурсий
1.15. `settings.py` — общие настройки системы
1.16. `slots.py` — управление имеющимися слотами (редактирование, перенос, назначение капитанов, удаление)
1.17. `statistic.py` — статистика и отчёты

### captain — роутер капитан-панели (с CaptainMiddleware)

2.1. `captain_main.py` — вход в капитан-панель (/captain), расписание, статистика, завершение экскурсии, отметка прибытия клиентов

### user — роутеры пользовательской панели

3.1. `user_main.py` — основные пользовательские команды, главное меню
3.2. `user_excursions.py` — просмотр экскурсий и расписания
3.3. `user_create_booking.py` — бронирование экскурсий пользователем
3.4. `user_payment.py` — оплата пользователем экскурсии
3.5. `account/main_router.py` — роутер личного кабинета
3.6. `account/adult_registration.py` — регистрация взрослого пользователя
3.7. `account/child_registration.py` — регистрация ребёнка
3.8. `account/token_registration.py` — регистрация по токену
3.9. `account/my_bookings.py` — мои бронирования
3.10. `account/personal_cabinet.py` — личный кабинет
3.11. `account/redaction_userdata.py` — редактирование данных пользователя
3.12. `account/redaction_childdata.py` — редактирование данных ребёнка

### fallback

4.1. `fallback.py` — обработка неизвестных команд и сообщений

## Правила именования

- Файлы: `{сущность}.py`
- Роутер: `{сущность}_router`
- Имя роутера: `router = Router(name="{сущность}")`

## Middleware

| Middleware | Назначение | Где применяется |
|------------|------------|-----------------|
| `AdminMiddleware` | Проверка роли `admin` | Все роутеры в `routers/admin/` |
| `CaptainMiddleware` | Проверка роли `captain` | Роутер `routers/captain/captain_main.py` |
