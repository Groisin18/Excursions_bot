# Telegram-бот для автоматизации экскурсионного бизнеса

Проект объединяет CRM, систему бронирования, финансовый учет и админ-панель, позволяя управлять всеми аспектами бизнеса напрямую из Telegram.

**Статус:** Реальный бизнес-проект в активной разработке.
**Разработчик:** Учебный проект одного разработчика в production-среде.

## Основные характеристики

*   Полнофункциональная админ-панель внутри Telegram
*   Интеграция с YooKassa для онлайн-оплаты
*   Система управления расписанием капитанов и экскурсий
*   Система логирования
*   Поддержка регистрации детей через токен-систему
*   Комплаенс с 152-ФЗ (согласие на обработку ПД)

## Возможности

### Для клиентов
*   Регистрация (полная или по токену для детей до 14 лет)
*   Личный кабинет с историей бронирований
*   Просмотр экскурсий с описанием и расписанием
*   Онлайн-бронирование с выбором даты и времени
*   Оплата через YooKassa (карты, СБП, кошельки)
*   Редактирование профиля

### Для администратора
*   Управление экскурсиями (создание, редактирование, деактивация)
*   Расписание (создание слотов, назначение капитанов, контроль загрузки)
*   Управление клиентами (поиск, редактирование, история)
*   Капитаны (назначение, график работы, зарплата)
*   Финансы (отслеживание платежей, расходы, отчетность)
*   Промокоды (создание, активация, статистика)
*   Уведомления (массовые и персональные рассылки)
*   Статистика (загрузка, доходы, популярные экскурсии)

## Технические особенности

## Установка и запуск

### Предварительные требования
*   Python 3.11 или новее
*   Telegram Bot Token от @BotFather
*   Тестовый или полноценный токен YooKassa

### Шаг 1: Настройка окружения
```bash
# Клонирование (пример)
cd C:\Users\admin\Desktop
# Создание виртуального окружения
python -m venv venv
venv\Scripts\activate  # Windows
# или
source venv/bin/activate  # Linux/Mac
```

### Шаг 2: Установка зависимостей
```bash
pip install -r requirements.txt
```

### Шаг 3: Конфигурация
Создайте файл `.env` в корне проекта:

```env
TG_TOKEN = ваш_токен_бота
PAYMENTS_TOKEN = ваш_токен_yookassa

# Логирование
LOG_LEVEL=INFO
ENABLE_CONSOLE_LOGGING=false
ENABLE_FILE_LOGGING=true
ROTATION_MAX_SIZE_MB=10
ROTATION_BACKUP_COUNT=5

REDIS_HOST= хост_редис
REDIS_PORT= порт_редис
REDIS_DB=0
REDIS_PASSWORD= пароль_редис (если не нужен, оставьте пустым)
```

### Шаг 4: Запуск
```bash
python run.py
```
База данных создастся автоматически при первом запуске.

## Бизнес-логика

### Ценообразование
*   Фиксированная цена за место на конкретный вид экскурсии
*   Нет дифференциации по возрасту, весу или количеству людей
*   Стоимость определяется в настройках экскурсии администратором

### Регистрация детей
*   Родитель регистрируется стандартным способом
*   Для ребенка генерируется токен (привязан к номеру родителя)
*   Токен используется для быстрой регистрации ребенка
*   При достижении 14 лет ребенок может активировать полный аккаунт, введя токен
*   Виртуальный телефон: `родительский_номер:токен:child`
    **Пример:** `+79512345678:1PLGuR:child`

### Регистрация через администратора
*   Администратор регистрирует человека, если тот по какой-то причине не может зарегистрироваться самостоятельно
*   Например, он не знает русского языка, или он уже пожилой и ему трудно пользоваться телеграмом
*   Такой человек не управляет своим аккаунтом, а записывается на экскурсии через администратора, оплачивая их затем наличными.
*   При желании такой человек может активировать полный аккаунт, введя токен

### Модели базы данных
Основные таблицы:
*   `users` - Пользователи системы
*   `excursions` - Экскурсии
*   `excursion_slots` - Слоты экскурсий
*   `bookings` - Бронирования
*   `bookingchild` - Привязка брони детей к брони родителя
*   `payments` - Платежи
*   `salary` - Зарплаты
*   `expense` - Траты
*   `promo_codes` - Промокоды
*   `notifications` - Уведомления
*   `telegramfile` - Хэши файлов в боте для отправки согласия на обработку ПД и иных

### Система ролей
*   `client` - Обычный пользователь
*   `captain` - Капитан (проводит экскурсии)
*   `admin` - Администратор (полный доступ к системе управления)

### Согласие на обработку ПД
*   Пользователь дает согласие внутри бота
*   Флаг согласия хранится в БД
*   Текст согласия соответствует требованиям 152-ФЗ
