# Telegram-бот для автоматизации экскурсионного бизнеса

Проект объединяет CRM, систему бронирования, финансовый учет, админ-панель и панель капитана, позволяя управлять всеми аспектами экскурсионного бизнеса напрямую из Telegram.

**Статус:** Production-тестирование. Основной функционал реализован, проект проходит проверку в реальных условиях эксплуатации.

**Разработчик:** Проект одного разработчика.

## Основные характеристики

*   Полнофункциональная админ-панель внутри Telegram
*   Панель капитана для управления расписанием и отметки прибытия клиентов
*   Интеграция с YooKassa для онлайн-оплаты
*   Система управления расписанием капитанов и экскурсий
*   Система логирования с ротацией файлов
*   Поддержка регистрации детей через токен-систему
*   Система возвратов средств
*   Автоматические уведомления и напоминания
*   Middleware для контроля доступа (admin, captain)
*   Комплаенс с 152-ФЗ (согласие на обработку ПД)

## Возможности

### Для клиентов

*   Регистрация (полная или по токену для детей до 14 лет)
*   Личный кабинет с историей бронирований
*   Просмотр экскурсий с описанием и расписанием
*   Онлайн-бронирование с выбором даты и времени
*   Оплата через YooKassa (карты, СБП, кошельки)
*   Отмена бронирования с автоматическим возвратом
*   Редактирование профиля и данных детей

### Для капитанов

*   Доступ к панели капитана по команде /captain
*   Просмотр личного расписания на неделю
*   Отметка прибытия клиентов на экскурсию
*   Завершение экскурсий
*   Просмотр личной статистики

### Для администратора

*   Управление экскурсиями (создание, редактирование, деактивация)
*   Расписание (создание слотов, назначение капитанов, контроль загрузки)
*   Управление клиентами (поиск, редактирование, история, создание через админа)
*   Капитаны (назначение, график работы, зарплата)
*   Финансы (отслеживание платежей, расходы, отчетность)
*   Промокоды (создание, активация, статистика)
*   Уведомления (массовые и персональные рассылки)
*   Статистика (загрузка, доходы, популярные экскурсии)
*   Управление возвратами средств
*   Управление ролями пользователей (/promote)

## Архитектура

Проект построен по модульной архитектуре с разделением ответственности:

*   app/routers/ — обработчики команд и callback-запросов (admin, captain, user)
*   app/database/ — слой работы с данными (репозитории, менеджеры, модели)
*   app/services/ — внешние сервисы (Redis, планировщик, YooKassa, уведомления)
*   app/middlewares/ — проверка прав доступа (AdminMiddleware, CaptainMiddleware)
*   app/admin_panel/ — клавиатуры и состояния для админ-панели
*   app/captain_panel/ — клавиатуры и состояния для панели капитана
*   app/user_panel/ — клавиатуры и состояния для панели пользователя
*   app/utils/ — вспомогательные модули (валидация, калькуляторы, логирование)
*   tests/ — модульные тесты

Принципы работы с данными:
*   UnitOfWork — для операций записи в базу данных (транзакционные изменения)
*   Репозитории — для CRUD-операций (прямое чтение)
*   Менеджеры — для сложной бизнес-логики

## Установка и запуск

### Предварительные требования

*   Python 3.11 или новее
*   Redis (для хранения состояний FSM)
*   Telegram Bot Token от @BotFather
*   Тестовый или полноценный токен YooKassa

### Шаг 1: Настройка окружения

# Клонирование репозитория
cd C:\Users\admin\Desktop
# или
cd /opt/excursions_bot

# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Linux/Mac)
source venv/bin/activate

### Шаг 2: Установка зависимостей

pip install -r requirements.txt

### Шаг 3: Конфигурация

Создайте файл .env в корне проекта:

TG_TOKEN = ваш_токен_бота
PAYMENTS_TOKEN = ваш_токен_yookassa

# Информация для возврата средств
YOOKASSA_SHOP_ID = ваш_shop_id
YOOKASSA_SECRET_KEY = ваш_секретный_ключ

# Уровень логирования
LOG_LEVEL = INFO # DEBUG, INFO, WARNING, ERROR, CRITICAL

# Куда писать логи
ENABLE_CONSOLE_LOGGING = false
ENABLE_FILE_LOGGING = true
LOG_DIR = logs

# Ротация логов
ROTATION_MAX_SIZE_MB = 10
ROTATION_BACKUP_COUNT = 5

# Redis
REDIS_HOST = localhost
REDIS_PORT = 6379
REDIS_DB = 0
REDIS_PASSWORD = пароль_редис (если не нужен, оставьте пустым)

# За сколько часов до начала экскурсии невозможен возврат денег
REFUND_HOURS_BEFORE = 4

### Шаг 4: Запуск

python run.py

База данных создастся автоматически при первом запуске.

## Важно! Часовой пояс сервера

Все даты в системе на момент разработки хранятся в местном времени Иркутска (Asia/Irkutsk, UTC+8). Если вы запускаете бота на новом сервере, или если вы переезжаете на сервер в другом часовом поясе, то старые данные в базе начнут отставать по времени. Поэтому при развертывании на новом сервере необходимо синхронизировать часовой пояс со старым.

### При развертывании на новом сервере:

# Установить часовой пояс Иркутска
sudo timedatectl set-timezone Asia/Irkutsk

# Проверить
date
# Должно показать время с +08

## Конфигурация

### Логирование

Система создает логи в директории logs/:

*   logs/app.log — основной файл логов
*   logs/errors.log — дублирование ошибок

Настройки логирования задаются в .env:

*   LOG_LEVEL — уровень логирования (DEBUG, INFO, WARNING, ERROR, CRITICAL)
*   ENABLE_CONSOLE_LOGGING — вывод логов в консоль (true/false)
*   ENABLE_FILE_LOGGING — запись логов в файлы (true/false)
*   ROTATION_MAX_SIZE_MB — максимальный размер файла лога в МБ
*   ROTATION_BACKUP_COUNT — количество файлов ротации

Рекомендуемые настройки для production:
*   10 МБ на файл, 5 бэкапов = максимум 60 МБ на каждый тип лога

Для высоконагруженных систем:
*   50 МБ на файл, 10 бэкапов

## Бизнес-логика

### Ценообразование

*   Фиксированная цена за место на конкретный вид экскурсии
*   Нет дифференциации по возрасту, весу или количеству людей
*   Стоимость определяется в настройках экскурсии администратором

### Регистрация детей

*   Родитель регистрируется стандартным способом
*   Для ребенка генерируется токен (привязан к номеру родителя)
*   Токен используется для быстрой регистрации ребенка
*   При достижении 14 лет ребенок может активировать полный аккаунт, введя токен
*   Виртуальный телефон: родительский_номер:токен:child
    Пример: +79512345678:1PLGuR:child

### Регистрация через администратора

*   Администратор регистрирует человека, если тот по какой-то причине не может зарегистрироваться самостоятельно
*   Например, он не знает русского языка, или он уже пожилой и ему трудно пользоваться телеграмом
*   Такой человек не управляет своим аккаунтом, а записывается на экскурсии через администратора, оплачивая их затем наличными
*   При желании такой человек может активировать полный аккаунт, введя токен

### Модели базы данных

Основные таблицы:

*   users — пользователи системы
*   excursions — экскурсии
*   excursion_slots — слоты экскурсий
*   bookings — бронирования
*   booking_children — привязка брони детей к брони родителя
*   payments — платежи
*   promo_codes — промокоды
*   notifications — уведомления
*   telegramfile — хэши файлов для отправки согласия на обработку ПД и иных

### Система ролей

*   client — обычный пользователь
*   captain — капитан (проводит экскурсии, имеет доступ к панели капитана)
*   admin — администратор (полный доступ к системе управления)

### Согласие на обработку ПД

*   Пользователь дает согласие внутри бота
*   Флаг согласия хранится в БД
*   Текст согласия соответствует требованиям 152-ФЗ
